# Obseri

Obseri gives a website knowledge, personality, voice, memory, and presence.

Enter a public URL and Obseri builds a source-grounded knowledge base. Shape the resulting Soul in Studio, talk to it by text or voice, then publish it as an isolated website widget. Conversation and lead events can be delivered to customer systems through signed webhooks.

## What works locally

- Bounded multi-page website ingestion with same-origin discovery and content deduplication
- Readable evidence chunks, source URLs, keyword extraction, crawl errors, and refresh
- Retrieval-grounded conversation with citations and a deterministic no-key fallback
- Optional OpenAI-compatible chat provider
- Personality name, purpose, role, tone, traits, greeting, instructions, and guardrails
- Browser speech recognition and speech synthesis
- Optional Voicebox profiles, speech streaming, and consent-gated cloning
- Responsive glass widget generated by `public/obseri-widget.js`
- Runtime publishing and widget conversation events
- HMAC-SHA256 webhooks with timestamps, event IDs, and idempotency keys
- Browser-local Studio persistence and JSON export
- Production Supabase/Postgres model for accounts, Souls, knowledge, conversations, voice consent, jobs, and webhook delivery

## Run

```powershell
npm.cmd install
npm.cmd run dev -- --host 127.0.0.1
```

Open:

- Landing: `http://127.0.0.1:8080/`
- Soul Studio: `http://127.0.0.1:8080/app`
- Ingestion health: `http://127.0.0.1:8080/api/ingest`
- Chat health: `http://127.0.0.1:8080/api/chat`

Founder mode needs no environment variables. Copy `.env.example` to connect hosted services.

## Voicebox (optional)

Obseri integrates with Voicebox through its REST service; Voicebox remains a separate sibling service and repository.

```powershell
cd ..\..\..\voicebox
docker compose up -d --build
# This checkout maps Voicebox's internal 17493 port to 127.0.0.1:17600.
```

Set `OBSERI_VOICEBOX_URL=http://127.0.0.1:17600`. Obseri loads Voicebox's Kokoro and Qwen preset catalogues, creates preset profiles on first use, and keeps browser voices as an offline fallback. Voice cloning requires a recorded rights basis (`self`, `permission`, or `licensed`) and affirmative consent.

## Honest boundary

The local build is a functional founder product and integration demo. Studio state is browser-local and published Souls live in the current server process, so a restart clears published runtime copies. Before public paid accounts, connect the included production schema, authentication, durable object/database storage, queues, rate limiting, domain verification, billing, observability, and an isolated browser crawler for JavaScript-heavy sites.

See [architecture](docs/architecture.md) and the [launch plan](docs/launch-plan.md).
